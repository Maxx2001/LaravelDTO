## Welkom

Welkom bij de documentatie van **Laravel DTO** - een uitgebreide demonstratie van Domain Driven Design (DDD) architectuur met Spatie Laravel Data voor type-safe DTOs in Laravel.

Dit project toont hoe je een schone, onderhoudbare Laravel applicatie bouwt met:
- **Domain-Driven Design** - Scheiding van business logica en framework code
- **Type-Safe DTOs** - Gebruik van Spatie Laravel Data voor bulletproof data handling
- **Domain Actions** - Geëncapsuleerde business logica met single responsibility
- **Modern Frontend** - Vue.js 3 + TypeScript + Inertia.js + Tailwind CSS

## Architectuur

Dit project gebruikt een **Domain-Driven Design** architectuur waarbij business logica gescheiden is van het framework. Dit zorgt voor:

- **Betere testbaarheid** - Business logica is onafhankelijk van Laravel
- **Herbruikbaarheid** - Domain code kan in andere contexten gebruikt worden
- **Duidelijkheid** - Business regels zijn expliciet gedocumenteerd
- **Onderhoudbaarheid** - Wijzigingen in business logica hebben minimale impact op framework code

### Request Flow

<Steps>
  <Step title="Controller">
    Ontvangt HTTP request en valideert input via DTO
  </Step>
  <Step title="DTO (Data Transfer Object)">
    Valideert en transformeert request data naar type-safe objecten
  </Step>
  <Step title="Domain Action">
    Voert business logica uit in geïsoleerde, testbare classes
  </Step>
  <Step title="Response">
    Retourneert response via Inertia of JSON API
  </Step>
</Steps>
